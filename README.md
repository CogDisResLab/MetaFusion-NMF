# MetaFusion-NMF: Cross-Cohort Multi-Omics Integration

This pipeline performs **Vertical Integration** (fusing mRNA, Methylation, and miRNA within a cohort) and **Horizontal Integration** (identifying conserved biological subtypes across different cohorts, such as Humans vs. iPSCs).

## How it Works

1. **Vertical Step (SNF):** Uses Similarity Network Fusion to create a single patient-patient similarity matrix from multiple omics layers.
2. **Stability Step (NMF):** Decomposes the fused network using Non-negative Matrix Factorization across multiple ranks ($k=2$ to $k=5$).
3. **Meta-Rank Selection:** Calculates the Cophenetic Correlation Coefficient (CCC) for every cohort and selects the rank that is most stable across the entire study.
4. **Horizontal Step (Meta-Projection):** Identifies **Conserved Drivers**—biomarkers that define the same subtype in both cohorts (e.g., Human Disease Subtype 1 vs. iPSC Model State 1).

## Installation

### 1. Prerequisites
Ensure you have **Conda** or **Mamba** installed.

### 2. Clone and Setup
```bash
git clone [https://github.com/your-username/meta-snf-nmf.git](https://github.com/your-username/meta-snf-nmf.git)
cd meta-snf-nmf
# The pipeline manages its own environment via Snakemake
# but you can preview the requirements in envs/environment.yaml
```
## Data Preparation
Organize your data into cohort-specific subdirectories within the data/ folder.

**CRITICAL: This pipeline expects Wide Format (ML Layout).**

Rows: Samples/Patients

Columns: Features (Gene Symbols, CpG Sites, miRNA IDs)

Feature names must be consistent across cohorts to enable Horizontal Integration.
```
data/
├── Cohort_A/
│   ├── expression.csv
│   ├── methylation.csv
│   └── mirna.csv
└── Cohort_B/
    ├── expression.csv
    ├── methylation.csv
    └── mirna.csv
```

## Configuration
Edit config.yaml to point to your specific files and set your desired NMF ranks:

```
cohorts:
  - "Humans"
  - "iPSCs"

data:
  Humans:
    mrna: "data/Humans/expression.csv"
    methy: "data/Humans/methylation.csv"
    mirna: "data/Humans/mirna.csv"
  iPSCs:
    mrna: "data/iPSCs/expression.csv"
    methy: "data/iPSCs/methylation.csv"
    mirna: "data/iPSCs/mirna.csv"
```

## Execution
Run the full pipeline using Snakemake. The --use-conda flag ensures all dependencies (SNFtool, Scikit-Learn, Seaborn) are handled automatically.

```
# Dry run to check logic
snakemake -n

# Execute with 4 cores
snakemake --use-conda --cores 4
```

# Pipeline Outputs and Requirements

## Key Outputs

| File | Description |
| :--- | :--- |
| `results/FINAL_META_REPORT.md` | **Executive Summary:** Contains the mathematical best rank (k), stability metrics, and the top 10 conserved genes for each cluster. |
| `results/meta_similarity_heatmap.png` | **Validation Heatmap:** A visual correlation matrix comparing Human subtypes against iPSC states to identify biological matches. |
| `results/meta_conserved_drivers.csv` | **Full Biomarker List:** A comprehensive CSV of every feature (gene, CpG, miRNA) that is conserved across cohorts. |
| `results/{cohort}/fused_network.npy` | **Fused Matrix:** The cohort-specific "Vertical" fused similarity matrix generated by the SNF algorithm. |

## Technical Requirements

### Core Environment
* **Python**: 3.9+
* **Workflow Engine**: Snakemake

### Machine Learning & Mathematics
* **Scikit-Learn**: Specifically for the `NMF` decomposition and `nndsvd` initialization.
* **SNFtool**: For the cross-modality similarity fusion and message passing.
* **Pandas & Numpy**: For data alignment, matrix manipulation, and I/O.

### Visualization & Reporting
* **Seaborn**: For generating the cross-cohort correlation heatmaps.
* **Matplotlib**: Core plotting engine for result visualization.
